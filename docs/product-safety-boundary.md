# Health Guidance 제품 임상 운영 경계 v0.2

- 정책 ID: `INTERNAL_PRODUCT_BOUNDARY_V0_2`
- 기준일: 2026-07-23
- 상태: `ACTIVE`
- 전제: 의료 자문, 임상 검수, 규제·개인정보 검토, 운영 모니터링을 확보할 수 있음
- 목적: 의료 기능을 원천 금지하지 않고 승인된 근거·프로토콜·입력 범위 안에서 안전하게 활성화

## 제품 포지션

이 앱은 일반 건강관리부터 개인 건강기록 해석, 질환 위험 평가, 검진 결과 해석, 질환별 운동,
복약 의사결정 지원, 긴급도 분류까지 확장 가능한 Health Guidance 플랫폼입니다. LLM은 독립적인
의학 기준을 만들지 않으며, 의료진이 승인한 정책과 근거를 실행하고 사용자에게 설명합니다.

## 목표 기능 범위

| 영역 | 지원 가능한 출력 | 활성화 조건 |
|---|---|---|
| 일반 운동 | 시간·장소·기구·선호도 기반 운동 구성 | 공식 근거와 일반 정책 검증 |
| 질환별 운동 | 질환·증상·약물·제한사항을 반영한 강도·종류·중단 기준 | 해당 질환 프로토콜과 의료진 승인 |
| 검진 | 값·단위·추세·판정·위험층화·권장 후속조치 | 검진일 기준 규칙, 필수 임상 맥락, 승인된 후속조치 표 |
| 증상·질환 | 감별 가능성, 위험 신호, 다음 행동, 진료과·시점 안내 | 증상별 프로토콜, 긴급도 규칙, 의료진 승인 |
| 복약 | 품목정보, 중복·상호작용·금기 확인, 복약 의사결정 지원 | 정확한 약물 식별, 전체 복약·알레르기·상태 확인, 승인된 약물 지식원 |
| 치료 지원 | 승인된 범위의 치료 옵션·모니터링·재검 계획 설명 | 질환별 진료지침과 의료진 승인, 사용자에게 한계·대안 표시 |
| 응급도 분류 | 즉시 대응·응급실·당일 진료·예약 진료·자가관리 분기 | 별도 고위험 프로토콜, 보수적 fail-safe, 운영 모니터링 |

## 핵심 원칙

의료 기능은 모두 제품 범위에 포함할 수 있습니다. 다만 “가능하다”는 것은 LLM이 자유롭게
추론해도 된다는 뜻이 아니라 다음 네 조건을 충족하면 정해진 출력 수준까지 제공할 수 있다는
뜻입니다.

1. 적용할 임상 프로토콜의 버전이 고정되어 있습니다.
2. 해당 프로토콜과 평가셋이 권한 있는 의료진에게 승인되어 있습니다.
3. 프로토콜이 요구하는 사용자 입력이 확인되어 있습니다.
4. 근거, 정책, 모델, 응답의 버전과 실행 로그를 추적할 수 있습니다.

## 임상 승인 게이트

고위험 기능은 다음 상태를 거칩니다.

```text
DESIGNED
  → READY_FOR_CLINICAL_REVIEW
  → CLINICALLY_APPROVED
  → ACTIVE
  → EXPIRED | SUSPENDED | REVOKED
```

승인은 기능 전체에 포괄적으로 부여하지 않습니다. `대상군 + 입력 조건 + 허용 출력 + 근거
버전 + 정책 버전 + 유효기간`의 조합에 부여합니다. 하나라도 범위를 벗어나면 그 요청에는
승인을 재사용하지 않습니다.

## 런타임 fail-safe

승인되지 않았거나 입력이 부족한 경우에도 기능 자체를 영구 금지하지 않습니다.

| 실패 원인 | 런타임 동작 |
|---|---|
| 필수 정보 부족 | 필요한 항목만 추가 질문 |
| 승인된 프로토콜 없음 | 의료진 검토 경로 또는 일반 정보로 전환 |
| 근거 부족·충돌 | 불확실성을 표시하고 단정하지 않음 |
| 고위험 신호 | 일반 답변을 중단하고 승인된 긴급도 프로토콜 실행 |
| 약물 식별 불확실 | 품목을 재확인하기 전 약물 판단 중단 |
| OCR 미확인 | 사용자가 원문과 대조하기 전 임상 판단 입력으로 사용하지 않음 |

## 사용자 데이터 원칙

- 사용자 건강기록은 운영용 임상 데이터 저장소에 암호화해 보관합니다.
- 사용자 원문과 대화는 공용 지식 VDB에 임베딩하지 않습니다.
- 개인화는 필요한 최소 데이터만 정책 엔진에 전달합니다.
- 임상 답변에 사용된 값, 출처, 정책, 모델 버전을 감사 로그로 남깁니다.
- 정정·삭제·동의 철회와 보관기간 정책을 데이터 계층에 반영합니다.

## 근거 사용 원칙

1. 공식 법령, 국내외 진료지침, 공식 의약품 정보, 체계적 문헌을 출처 계층에 따라 관리합니다.
2. 수치 판정과 금기처럼 정확성이 필요한 규칙은 관계형·규칙형 DB로 실행합니다.
3. VDB는 설명 근거 검색에 사용하며 정책 승인이나 수치 판정을 대신하지 않습니다.
4. 의료진이 만든 결론도 적용 대상, 예외, 중단 기준과 출처를 함께 기록합니다.
5. 서로 충돌하는 지침은 우선순위와 관할, 발행일, 대상군을 의료진이 명시적으로 결정합니다.

## 의료진 검수 범위

의료진은 최소한 다음 산출물을 검토합니다.

- Evidence의 원문 충실도와 적용 대상
- 수치 기준, 예외, 금기, 중단·전원 기준
- Policy 조건식과 우선순위
- 사용자 응답 템플릿과 불확실성 표현
- 일반·경계·반례·위험 사례를 포함한 골든 평가셋
- 변경 전후 회귀 테스트와 유효기간

## 별도 검토 트랙

의료 자문이 있다는 전제와 별개로 개인정보, 보안, 의료기기·의료서비스 해당 여부, 책임 주체,
운영 중 사고 대응은 각각 별도 검토합니다. 면책문구는 임상 승인과 기술적 통제를 대신하지
않습니다.
