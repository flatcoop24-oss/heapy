# 임상 거버넌스 및 의료진 검수 절차 v0.2

## 1. 역할

| 역할 | 책임 |
|---|---|
| Clinical Owner | 도메인 범위, 최종 승인, 중단·철회 결정 |
| Medical Reviewer | Evidence·규칙·예외·사용자 표현 검토 |
| Pharmacist Reviewer | 약물 식별, 상호작용, 금기, 복약 출력 검토 |
| Exercise Specialist | 질환별 운동 강도·금기·중단 기준 검토 |
| Data Steward | 출처·라이선스·버전·계보 관리 |
| ML/LLM Engineer | 검색·라우팅·출력 검증·평가 구현 |
| Safety Operator | 운영 모니터링, 사고 대응, capability 중지 |

한 사람이 여러 역할을 맡을 수 있지만 작성자와 최종 승인자는 고위험 기능에서 분리합니다.

## 2. 승인 패킷

capability 하나를 승인하려면 다음 자료가 함께 있어야 합니다.

1. 적용 대상과 제외 대상
2. 질문 의도와 필수 입력 스키마
3. 공식 출처와 Evidence 목록
4. 정확 규칙, 예외, 금기, 중단·전원 조건
5. Policy 우선순위와 충돌 해결 규칙
6. 사용자 응답 템플릿과 불확실성 표현
7. 정상·경계·반례·결측·위험 사례 골든셋
8. 성능 기준, 위험 허용 한계, 운영 모니터링 지표
9. 승인자 역할, 승인일, 유효기간, 재검토 조건

## 3. 출처 계층

```text
법령·정부 고시·공식 허가정보
  → 국내 전문학회·공공기관 진료지침
  → 국제 가이드라인·체계적 문헌
  → 동료심사 연구
  → 전문가 합의문
```

출처 등급만으로 자동 승인하지 않습니다. 최신성, 대상군, 관할, 근거 강도, 다른 지침과의
충돌을 검토자가 함께 판정합니다.

## 4. 승인 판정

- `APPROVED`: 지정한 범위에서 운영 가능
- `CHANGES_REQUESTED`: 수정 후 재검토
- `REJECTED`: 근거 또는 위험 통제가 불충분
- `EXPIRED`: 유효기간 만료
- `REVOKED`: 안전 이슈·지침 변경 등으로 즉시 철회

승인은 capability 이름 하나가 아니라 `대상군 + 정책 버전 + Evidence 버전 + 모델/프롬프트
계약 + 출력 수준`에 귀속됩니다.

## 5. 변경 관리

다음 변경은 재승인을 요구합니다.

- 수치 임계치, 금기, 긴급도, 후속조치 변경
- 적용 대상 또는 허용 출력 확대
- 핵심 지침의 신규 버전 반영
- 검색 필터나 정책 우선순위 변경
- 모델 교체로 의료 claim 생성 양상이 달라지는 경우
- 골든셋 안전성 기준 미달 또는 운영 사고 발생

문장 교정처럼 임상 의미가 변하지 않는 변경은 원문 충실도 검수로 처리할 수 있지만 변경
로그는 남깁니다.

## 6. 배포 게이트

```text
정적 JSONL 검증
  → 정책 단위 테스트
  → 임상 골든셋 평가
  → 적대·결측·충돌 테스트
  → 의료진 승인 확인
  → shadow mode
  → 제한된 운영 활성화
  → 모니터링 및 주기 재검토
```

배포 도구는 `CLINICALLY_APPROVED` 상태, 유효기간, 정책·Evidence 버전 일치를 확인하지 못하면
고위험 capability를 빌드 산출물에 포함하지 않습니다.

## 7. 운영 중 통제

- capability 단위 kill switch
- 답변별 정책·근거·모델·입력 버전 감사 로그
- 위험 답변 신고와 의료진 재검토 큐
- 근거 없는 claim, 인용 오류, 금기 누락, 과소·과대 분류 모니터링
- 지침 개정과 승인 만료 알림
- 사고 발생 시 재현 가능한 입력·출력 스냅샷
